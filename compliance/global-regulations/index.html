    Global Regulatory Compliance Guide¶

    Document Version: 1.0 Last Updated: 2025-11-24 Status: Active

    Overview¶

    FraiseQL's security features support compliance with supply chain security regulations and industry standards across multiple jurisdictions. This document provides a comprehensive mapping of FraiseQL features to global regulatory requirements.

    üåç Jurisdiction-Specific Requirements¶

    üá∫üá∏ United States¶

    Executive Order 14028 (May 2021)¶

    "Improving the Nation's Cybersecurity"

    Requirements: - Software vendors must provide Software Bill of Materials (SBOM) - Use of secure software development practices - Supply chain security for federal procurement

    FraiseQL Support: - ‚úÖ Automated SBOM generation (fraiseql sbom generate) - ‚úÖ CycloneDX 1.5 format (OWASP standard) - ‚úÖ Cryptographic signing with Cosign - ‚úÖ CI/CD integration for automated compliance

    Reference: White House EO 14028

    NIST SP 800-161 Rev. 1¶

    Cybersecurity Supply Chain Risk Management

    Requirements: - Identify and assess supply chain risks - Implement risk mitigation strategies - Continuous monitoring and assessment

    FraiseQL Support: - ‚úÖ SBOM provides complete dependency visibility - ‚úÖ Package URL (PURL) identifiers for vulnerability tracking - ‚úÖ Cryptographic hashes for integrity verification

    Reference: NIST SP 800-161

    NIST SP 800-218¶

    Secure Software Development Framework (SSDF)

    Requirements: - Prepare the Organization (PO) - Protect the Software (PS) - Produce Well-Secured Software (PW) - Respond to Vulnerabilities (RV)

    FraiseQL Support: - ‚úÖ Security profiles (STANDARD, REGULATED, RESTRICTED) - ‚úÖ Multi-provider KMS for key management - ‚úÖ Observability with OpenTelemetry - ‚úÖ SBOM for vulnerability response

    Reference: NIST SP 800-218

    CCCS SBOM Guidance¶

    Canadian Centre for Cyber Security

    Requirements: - Joint guidance with US CISA on SBOM adoption - Software transparency for critical infrastructure

    FraiseQL Support: - ‚úÖ CycloneDX/SPDX format support - ‚úÖ Automated SBOM generation - ‚úÖ Integration with vulnerability databases

    Reference: CCCS SBOM Guidance

    Canadian Program for Cyber Security Certification (CPCSC)¶

    Effective: March 2025 (phased through 2027)

    Requirements: - Cyber security certification for defence contractors - Self-assessment and third-party audits - Continuous compliance monitoring

    FraiseQL Support: - ‚úÖ RESTRICTED security profile for defence applications - ‚úÖ Audit logging and compliance reporting - ‚úÖ KMS integration for cryptographic requirements

    Reference: CPCSC Program

    üá™üá∫ European Union¶

    NIS2 Directive (Directive 2022/2555)¶

    Network and Information Systems Directive Effective: October 2024

    Requirements: - Supply chain security risk management - Incident reporting (including supply chain incidents) - Security measures for essential and important entities

    Sectors Covered: - Energy, transport, healthcare, finance, water, digital infrastructure, manufacturing, postal services, public administration, space

    FraiseQL Support: - ‚úÖ Supply chain transparency via SBOM - ‚úÖ Security event logging and audit trails - ‚úÖ Incident detection and reporting capabilities

    Reference: NIS2 Directive (EU)

    EU Cyber Resilience Act (CRA)¶

    Phasing in: 2025-2027 üî• Explicit SBOM Requirement

    Requirements: - Manufacturers must create and maintain SBOM in machine-readable format - Must include top-level dependencies - Update SBOM with each release - Vulnerability disclosure process

    Products Covered: - All products with software components sold in EU

    FraiseQL Support: - ‚úÖ Explicit SBOM compliance - CycloneDX 1.5 - ‚úÖ Direct and transitive dependencies included - ‚úÖ Automated CI/CD generation - ‚úÖ Version-tracked SBOMs

    Reference: EU CRA SBOM Requirements

    üöÄ Implementation Roadmap¶

    Phase 1: Assessment¶

    1. Identify applicable regulations for your jurisdiction
    2. Determine required security profile (STANDARD/REGULATED/RESTRICTED)
    3. Review SBOM requirements

    Phase 2: Configuration¶

    1. Enable security profile: 
      from fraiseql.security.profiles import SecurityProfile, ProfileEnforcer

enforcer = ProfileEnforcer(profile=SecurityProfile.REGULATED)
    2. Configure KMS provider (if required): 
      from fraiseql.security.kms import VaultKMSProvider, VaultConfig

kms = VaultKMSProvider(VaultConfig(
    vault_addr="https://vault.example.com:8200",
    token=os.environ["VAULT_TOKEN"]
))

    Phase 3: SBOM Generation¶

    1. Generate SBOM: 
      fraiseql sbom generate --output fraiseql-1.0.0-sbom.json
    2. Integrate into CI/CD (see .github/workflows/sbom-generation.yml)
    3. Distribute SBOM to customers/auditors

    Phase 4: Continuous Compliance¶

    1. Enable audit logging
    2. Monitor security events
    3. Update SBOM with each release
    4. Conduct regular security assessments
