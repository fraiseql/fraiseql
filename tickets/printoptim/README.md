# PrintOptim Team Tickets

This directory contains tickets for the PrintOptim team's work on FraiseQL.

## Directory Structure

```
printoptim/
├── active/          # Currently in-progress tickets
├── completed/       # Completed tickets (for reference)
└── README.md        # This file
```

## Ticket Naming Convention

Use the following format for ticket files:
- `TICKET_<number>_<brief_description>.md`
- Example: `TICKET_001_query_performance_optimization.md`

## Ticket Template

```markdown
# TICKET_XXX: [Title]

**Status:** Active/In Progress/Completed
**Priority:** High/Medium/Low
**Assigned to:** [Team member name]
**Created:** YYYY-MM-DD
**Updated:** YYYY-MM-DD

## Description
[Detailed description of the issue or feature]

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Technical Details
[Any relevant technical information]

## Notes
[Additional notes, updates, or comments]
```

## Workflow

1. Create new tickets in the `active/` directory
2. Update ticket status and content as work progresses
3. Move completed tickets to `completed/` directory
4. Keep ticket files for future reference and documentation

## Integration with GitHub Issues

For tickets that need external visibility or collaboration:
- Create a corresponding GitHub issue
- Reference the GitHub issue number in the ticket file
- Link back to the internal ticket in the GitHub issue description
