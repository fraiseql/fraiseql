# FraiseQL Configuration Reference

All FraiseQL configuration lives in `fraiseql.toml`. Security and operational settings are compiled into `schema.compiled.json` by `fraiseql-cli compile`, then loaded by the server at startup. Environment variables can override compiled settings in production.

## Project

```toml
[project]
name = "my-fraiseql-app"
version = "1.0.0"
description = "My GraphQL API"
```

## Schema Paths

```toml
[fraiseql]
schema_file = "schema.json"            # Input: generated by Python/TS decorators
output_file = "schema.compiled.json"   # Output: compiled schema + config
```

## Security

### Audit Logging

```toml
[fraiseql.security.audit_logging]
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `true` | Enable audit logging |
| `log_level` | string | `"info"` | Log level: `"debug"`, `"info"`, `"warn"` |
| `include_sensitive_data` | bool | `false` | Include PII in logs (never in production) |
| `async_logging` | bool | `true` | Buffer events for async flush |
| `buffer_size` | u32 | `1000` | Events to buffer before flushing |
| `flush_interval_secs` | u32 | `5` | Seconds between flushes |

**Env overrides:** `AUDIT_LOG_LEVEL`

### Error Sanitization

```toml
[fraiseql.security.error_sanitization]
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `true` | Enable error sanitization |
| `generic_messages` | bool | `true` | Replace detailed errors with generic messages |
| `internal_logging` | bool | `true` | Log full error details internally |
| `leak_sensitive_details` | bool | `false` | Expose implementation details (never in production) |
| `user_facing_format` | string | `"generic"` | Format: `"generic"`, `"simple"`, `"detailed"` |

### Rate Limiting

```toml
[fraiseql.security.rate_limiting]
```

| Field | Type | Default | Env Override | Description |
|-------|------|---------|--------------|-------------|
| `enabled` | bool | `true` | | Enable rate limiting |
| `auth_start_max_requests` | u32 | `100` | `RATE_LIMIT_AUTH_START` | Max /auth/start per IP per window |
| `auth_start_window_secs` | u64 | `60` | | Window duration |
| `auth_callback_max_requests` | u32 | `50` | `RATE_LIMIT_AUTH_CALLBACK` | Max /auth/callback per IP per window |
| `auth_callback_window_secs` | u64 | `60` | | Window duration |
| `auth_refresh_max_requests` | u32 | `10` | `RATE_LIMIT_AUTH_REFRESH` | Max /auth/refresh per user per window |
| `auth_refresh_window_secs` | u64 | `60` | | Window duration |
| `auth_logout_max_requests` | u32 | `20` | `RATE_LIMIT_AUTH_LOGOUT` | Max /auth/logout per user per window |
| `auth_logout_window_secs` | u64 | `60` | | Window duration |
| `failed_login_max_requests` | u32 | `5` | `RATE_LIMIT_FAILED_LOGIN` | Max failed attempts before lockout |
| `failed_login_window_secs` | u64 | `3600` | | Lockout window (1 hour) |

### State Encryption (PKCE)

```toml
[fraiseql.security.state_encryption]
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `true` | Enable PKCE state encryption |
| `algorithm` | string | `"chacha20-poly1305"` | Encryption algorithm (only supported value) |
| `key_rotation_enabled` | bool | `false` | Future: automatic key rotation |
| `nonce_size` | u32 | `12` | Nonce size in bytes (immutable) |
| `key_size` | u32 | `32` | Key size in bytes (immutable) |

**Env override (required):** `STATE_ENCRYPTION_KEY` (base64-encoded 32-byte key)

Generate with: `openssl rand -base64 32`

### Constant-Time Comparison

| Field | Type | Default | Description |

| `apply_to_csrf_tokens` | bool | `true` | CSRF token validation |

## MCP (Model Context Protocol)

```toml
[mcp]
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | bool | `false` | Enable MCP server |
| `transport` | string | `"http"` | Transport: `"http"`, `"stdio"`, `"both"` |
| `path` | string | `"/mcp"` | HTTP path for MCP endpoint |
| `require_auth` | bool | `true` | Require authentication for MCP |
| `include` | string[] | `[]` | Whitelist of operations (empty = all) |
| `exclude` | string[] | `[]` | Blacklist of operations |

**Env trigger:** Set `FRAISEQL_MCP_STDIO=1` to enable stdio transport mode.

## Deployment Presets

### Development

```toml
[fraiseql.security.audit_logging]
log_level = "debug"
include_sensitive_data = true

[fraiseql.security.error_sanitization]
user_facing_format = "detailed"

[fraiseql.security.rate_limiting]
auth_start_max_requests = 10000
failed_login_max_requests = 10000
```

### Production

```toml
[fraiseql.security.audit_logging]
log_level = "info"
include_sensitive_data = false

[fraiseql.security.error_sanitization]
generic_messages = true
leak_sensitive_details = false

[fraiseql.security.rate_limiting]
auth_start_max_requests = 100
failed_login_max_requests = 5
```

### Enterprise

```toml
[fraiseql.security.audit_logging]
log_level = "debug"

flush_interval_secs = 1

[fraiseql.security.rate_limiting]

```
